# Sitemap & Robots Automation

## sitemap.xml

Currently generated by a server route at `src/routes/sitemap[.]xml.ts` (if present) or as a static `public/sitemap.xml` file.

### How Generation Works

1. The route enumerates static routes from `src/routeTree.gen.ts`.
2. Adds one entry per visible item from dynamic content (blog posts, projects, etc.).
3. Outputs an XML document with `<loc>`, `<lastmod>`, `<changefreq>`, `<priority>`.
4. Visibility-filtered: `visible: false` and `archived: true` entries are excluded by default.

### How to Customize

- Change base URL: edit `BASE_URL` in the route file.
- Add a new route: re-export it from the `entries` array.
- Change priority/frequency: per-entry fields.

## robots.txt

Static file at `public/robots.txt`. Edits go directly there.

Default:
```
User-agent: *
Allow: /
Sitemap: https://example.com/sitemap.xml
```

### Common Edits

- Hide a path: add `Disallow: /private`.
- Block crawlers entirely: replace `Allow: /` with `Disallow: /`.
- Allow only Googlebot: add a dedicated `User-agent: Googlebot` block before `User-agent: *`.

## Build-Time Automation

The sitemap server route runs on every request — content stays in sync automatically. For purely static hosts that can't run server routes, regenerate the static `public/sitemap.xml` as part of `bun run build` via a pre-build script.
